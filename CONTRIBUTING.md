# 贡献指南

感谢您考虑为 OpenBridge 做出贡献！

## 开发流程

1. Fork 本仓库
2. 创建您的特性分支
3. 提交您的更改
4. 确保代码符合项目规范
5. 提交 Pull Request

## 代码规范

- 遵循 Lua 标准编码规范
- 保持代码简洁清晰
- 添加必要的注释
- 确保代码可测试

## 提交 Pull Request

1. 更新 CHANGELOG.md
2. 确保所有测试通过
3. 更新相关文档
4. 提交 Pull Request 时请详细描述更改内容

## 报告 Bug

使用 Issue 模板报告 bug，并提供：

- 问题描述
- 重现步骤
- 期望行为
- 环境信息
